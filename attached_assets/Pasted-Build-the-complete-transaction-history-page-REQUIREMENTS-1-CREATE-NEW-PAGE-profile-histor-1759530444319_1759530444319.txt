4. DATABASE QUERY:

API Endpoint: GET /api/transactions/user/:userId/history

Query params:
- ?status=all|completed|cancelled|noshow
- &role=all|buyer|seller
- &date=all|90days|12months

Return from offers table joined with:
- listings (for item details)
- users (for other party details)
- user_statistics (for ratings)
- reviews (for review text if exists)

Order by: created_at DESC

5. STATUS BADGES:

Map offer.status to display:
- "completed" → ✅ COMPLETED (green)
- "buyer_cancelled" → ❌ CANCELLED BY BUYER (red)
- "seller_cancelled" → ❌ CANCELLED BY SELLER (red)
- "buyer_noshow" → ⚠️ NO-SHOW BY BUYER (orange)
- "seller_noshow" → ⚠️ NO-SHOW BY SELLER (orange)
- "mutual_cancel" → ↔️ MUTUALLY CANCELLED (gray)
