ISSUE 1: VALIDATION ERROR REDIRECTS TO WRONG PAGE

Problem: When user doesn't fill required category field (red asterisk), validation fails and returns them to photo upload screen instead of keeping them on the form.

Fix:
- When "Create 3 Separate Listings" is clicked, validate ALL required fields BEFORE submission
- If any required field is missing, stay on the current form page
- Scroll to the first missing field
- Highlight the missing field in red
- Show error message: "Please select a category for all items"
- Don't redirect anywhere until all required fields are complete

Validation logic:
```javascript
const validateAllItems = (items) => {
  const errors = [];
  items.forEach((item, index) => {
    if (!item.category) {
      errors.push({
        itemIndex: index,
        field: 'category',
        message: `Item ${index + 1}: Please select a category`
      });
    }
    if (!item.title) {
      errors.push({
        itemIndex: index,
        field: 'title',
        message: `Item ${index + 1}: Please enter a title`
      });
    }
    // Check other required fields...
  });
  
  if (errors.length > 0) {
    // Show errors, scroll to first error, don't redirect
    showValidationErrors(errors);
    scrollToFirstError();
    return false;
  }
  return true;
};
